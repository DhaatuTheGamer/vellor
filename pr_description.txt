🎯 **What:** The FAB (Floating Action Button) component was missing tests. This PR adds test coverage for rendering, interactions (opening/closing the menu), and navigation logic (Add Student and Quick Log features). We also resolved a duplicate TypeScript export (`DEFAULT_USER_NAME`) in `constants.ts` that caused the vite build to fail locally during the test runs.

📊 **Coverage:**
- Verified the initial rendering of the main FAB button.
- Verified that clicking the main FAB toggles the child menu items (Quick Log, Add Student).
- Verified that clicking the main FAB twice successfully toggles and closes the menu.
- Verified that clicking "Add Student" triggers `react-router-dom`'s `useNavigate` to route to `/students`.
- Verified that clicking "Quick Log" successfully updates state to show the `QuickLogModal`.
- Verified that calling `onClose` inside `QuickLogModal` properly closes the modal.

✨ **Result:** Test coverage improved for UI component interactions, bringing `components/ui/FAB.tsx` under testing and safeguarding its routing functionality from future regressions.
