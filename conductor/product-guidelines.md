# Vellor Product Guidelines

## 1. UX Principles
Vellor strikes a balance between being a powerful productivity tool and a delightful experience:
- **App-Like Fluidity:** Interactions should feel native, relying on fluid animations (via Framer Motion) and robust offline-first functionality.
- **Utilitarian & Dense:** Data displays (like the dashboard and student lists) must prioritize information density and speed, minimizing necessary clicks.
- **Playful & Engaging:** Key actions (e.g., unlocking achievements, logging milestones) are rewarded visually using confetti and interactive elements.

## 2. Tone & Voice
The application's voice should be **Encouraging & Warm**.
- Provide supportive feedback for positive actions (e.g., "Great job adding your first student!").
- Empty states should offer clear, encouraging calls-to-action rather than dead ends.
- Keep the gamification elements fun but professional enough for daily use.

## 3. Design System
Vellor combines several visual approaches to create a cohesive whole:
- **Shape & Contrast:** Use rounded, soft corners for cards and friendly modals, while maintaining flat, minimal aesthetics for data-heavy sections.
- **Theming:** All new UI must be built with a **Dark Mode First** mentality, ensuring high contrast and readability across varied environments.
- **Components:** Rely on existing Radix UI primitives and Tailwind CSS utility classes to maintain consistency.
